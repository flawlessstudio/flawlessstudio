# Flawless Terminology

> Canonical glossary of all terms used across the Flawless ecosystem. When in doubt, refer here.

## Core Terms

| Term | Definition |
|---|---|
| **Canon** | The immutable set of documents that define the Flawless system |
| **Ecosystem** | The full network of Flawless repositories, tools, and agents |
| **Framework** | The operating principles and structural rules of Flawless Studio |
| **Flawless Filter** | The quality gate applied to every decision, artifact, and system |
| **INUI OS** | The design system operating system powering Flawless UI |
| **FDIS OS** | Flawless Design Intelligence System — AI-native design layer |
| **Flawless Engine** | Core automation and orchestration layer |
| **Registry** | A versioned catalog of active tools, stacks, or agents |
| **Runbook** | A step-by-step operational guide for a specific task or system |
| **Agent Contract** | Formal specification of an AI agent's scope, inputs, outputs, and constraints |
| **Diátaxis** | Documentation framework separating tutorials, how-to, explanation, and reference |
| **Bilingual pair** | A canonical document and its ES translation (`file.md` + `file.es.md`) |
