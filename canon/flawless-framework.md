# Flawless Framework

> The core operating framework of Flawless Studio. Defines how systems are designed, built, and evolved.

## Principles

1. **Flawless over fast** — Quality is non-negotiable. Speed is earned through clarity, not shortcuts.
2. **Single source of truth** — Every concept lives in one canonical place. No duplication of meaning.
3. **Fractal coherence** — Every layer of the system (file, folder, repo, ecosystem) follows the same structural logic.
4. **Bilingual by default** — All canonical documents exist in EN and ES. EN is authoritative.
5. **Executable architecture** — Structure is not decoration. Every folder, file, and naming convention must be actionable.
6. **Minimum viable surface** — Expose only what is necessary. Complexity is hidden, not eliminated.
7. **Iterative permanence** — Systems evolve through deliberate, documented iterations. Nothing breaks silently.

## System Layers

| Layer | Scope | Example |
|---|---|---|
| **Canon** | Immutable truth | `canon/flawless-framework.md` |
| **Registry** | Active state of tools/stacks | `registries/voice-ai/` |
| **Ecosystem** | Map of all repos and their relationships | `ecosystem/map.md` |
| **Docs** | Operational knowledge, decisions, runbooks | `docs/` |
| **Agents** | Automated systems operating under contracts | `docs/reference/` *(contracts added when formalized)* |

## Quality Bar

Every artifact produced under this framework must pass the **Flawless Filter**:
- Is it the simplest correct solution?
- Is it bilingual and documented?
- Is it traceable to a canonical source?
- Is it free of redundancy?
