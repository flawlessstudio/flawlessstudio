# Security Policy

## Scope

This policy applies to all repositories under the [`flawlessstudio`](https://github.com/flawlessstudio) organization. None of them contain user-facing executable code at this time, so there are no versioned releases with active security patches.

## Reporting a vulnerability

If you discover a security issue in any Flawless Studio repository or ecosystem artifact:

1. **Do not open a public issue.**
2. Email `oneflawlessstudio@gmail.com` with subject line `[SECURITY] <brief description>`.
3. Include: affected repository, description of the issue, and reproduction steps if applicable.
4. You will receive a response within 72 hours.

## What we address

- Credentials or secrets accidentally committed to any repo
- Malicious content introduced via pull request
- Supply chain risks in referenced tooling or dependencies
- Misconfigured permissions or access controls in public repositories

## Out of scope

- Theoretical vulnerabilities without a demonstrable impact path
- Issues in third-party tools not maintained by Flawless Studio
- Vulnerabilities in forked repositories not actively maintained here
