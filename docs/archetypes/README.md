# Flawless Method — Archetype Index

This directory contains the 20 canonical project archetypes of the Flawless Method.
Each archetype is a fully specified operational system covering identity, scope, phases,
quality controls, validation gates, resilience, international standards, and lock conditions.

---

## What is an archetype?

An archetype is a **project type classifier and operational template**.
It answers: *"Given this kind of project, what is the right structure, standard, and governance?"*

Every archetype includes:

| Layer | Description |
|---|---|
| **Identity** | Output type, novelty class, primary standard, sector |
| **Scope** | What it covers |
| **Exclusions** | What it does not cover and where to redirect |
| **Primary use case** | Concrete application examples |
| **Phase structure** | Ordered operational phases |
| **Quality controls** | Active controls per phase |
| **Validation gates** | No-return checkpoints with explicit criteria |
| **Resilience** | Recovery and contingency protocols |
| **International validity** | Applicable international standards |
| **Lock condition** | When the project closes and when it reopens |

---

## Novelty classes

| Class | Meaning |
|---|---|
| **Mark 1** | First-of-kind. Target state emergent. Path built as you go. |
| **Mark n** | Repeatable type. Format known. Variables differ per instance. |
| **Mark 1/n** | Hybrid. Standard-defined target, emergent pathway. |

## How to use archetypes

1. **Classify** your project using the identity and scope of each archetype.
2. **Select** the archetype whose output type and sector best match.
3. **Apply** the phase structure, quality controls, and validation gates.
4. **Adapt** — add context-specific controls without removing mandatory gates.
5. **Lock** when the lock condition is met.

If your project spans two archetypes, select the primary archetype by output type and import
the secondary archetype's gates as supplementary controls.
