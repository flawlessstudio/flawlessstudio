# Archetype 15 — Event Management

> **Flawless Method** | `docs/archetypes/` | v1.0.0 | 2026-05-25

---

## Identity

| Field | Value |
|---|---|
| **Output type** | Delivered event — conference, product launch, trade show, corporate event, or live experience |
| **Novelty class** | Mark n — event type known; format, scale, and experience variable |
| **Primary standard** | ISO 20121:2012 (sustainable events), APEX/ANSI event standards, H&S regulations |
| **Sector** | Corporate, Entertainment, Sports, Government, Education, Trade, Cultural |

---

## Scope

All activities from event concept and brief through venue, logistics, programme design, production, delivery, and post-event review. Covers in-person, virtual, and hybrid events.

---

## Exclusions

| Out of scope | Redirect to |
|---|---|
| Marketing campaign promoting the event | Archetype 14 |
| Permanent venue construction | Archetype 03 |
| Content series produced at the event | Archetype 14 |
| Technology platform for virtual event | Archetype 01 |

---

## Primary Use Cases

- Annual conference or summit (B2B or B2C)
- Product launch event
- Trade show presence or booth
- Corporate away day, team event, or town hall
- Hybrid or virtual event production

---

## Phase Structure

| # | Phase | Key outputs |
|---|---|---|
| 1 | **Brief & concept** | Event brief, objectives, audience, format, budget envelope |
| 2 | **Planning** | Venue selection, supplier contracts, programme outline, run of show |
| 3 | **Production** | AV, staging, catering, registration, speaker management, rehearsals |
| 4 | **Delivery** | Event executed, live issue log, on-site command structure active |
| 5 | **Post-event** | Attendee survey, KPI review, financial reconciliation, lessons learned |

---

## Quality Controls

- Event brief signed off before any supplier engagement
- Health & safety risk assessment completed and approved before go-live
- Run of show document finalised and distributed 48h before event
- All key suppliers confirmed with signed contracts before production phase
- Contingency budget ≥ 10% of total event budget held in reserve
- Technical rehearsal completed before doors open
- Attendee NPS / satisfaction survey deployed within 24h of event close

---

## Validation Gates

| Gate | Criteria | Who approves |
|---|---|---|
| **G1 — Brief** | Objectives, audience, format, and budget approved | Event Lead + Sponsor |
| **G2 — Planning complete** | Venue, key suppliers contracted, programme approved | Event Lead + Sponsor |
| **G3 — Production ready** | Run of show final, rehearsal done, H&S cleared | Event Lead + H&S Officer |
| **G4 — Go/no-go** | All critical dependencies confirmed, contingencies in place | Event Lead + Sponsor |
| **G5 — Post-event** | KPIs measured, survey deployed, budget reconciled | Event Lead + Finance |

---

## Resilience & Recovery

- Contingency venue identified for critical events
- Weather and force majeure clause in all venue and supplier contracts
- On-site incident response plan with named first-aider and emergency contacts
- Virtual/hybrid fallback plan for in-person events above 100 attendees
- Real-time issue log and escalation path during event delivery

---

## International Validity

| Standard | Scope |
|---|---|
| ISO 20121:2012 | Sustainable event management |
| APEX/ANSI Event Standards | US event industry standards |
| BS 8901:2009 | Sustainable events (UK) |
| H&S at Work Act / EU Directive | Health and safety at events |
| GDPR | Attendee data collection and processing |
| ADA / EAA | Accessibility at events |

---

## Lock Condition

**Locks when:** Post-event review complete, KPIs documented, budget reconciled, and lessons learned archived.

**Reopens when:** Next edition of the event enters planning, or a follow-on event is commissioned.
